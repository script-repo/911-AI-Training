ACCESSING THE APPLICATION:
--------------------------
Since Ingress is disabled, you can access the services using port-forward:

  Frontend:
  $ kubectl port-forward -n {{ .Values.global.namespace }} svc/frontend-service 3000:80
  Then access: http://localhost:3000

  Backend API:
  $ kubectl port-forward -n {{ .Values.global.namespace }} svc/backend-service 8000:8000
  Then access: http://localhost:8000/docs

IMPORTANT NOTES:
----------------
1. Make sure you have created the required secrets:
   - openrouter-secret
   - database-secret
   - s3-secret

   You can create them using:
   $ kubectl create secret generic openrouter-secret \
     --from-literal=OPENROUTER_API_KEY=your_key \
     -n {{ .Values.global.namespace }}

2. Update the Ingress host in values.yaml to match your domain

3. For production, enable TLS/SSL in the Ingress configuration

4. Monitor resource usage and adjust requests/limits as needed

TROUBLESHOOTING:
